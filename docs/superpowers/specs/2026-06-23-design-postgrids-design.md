# 画面设计台 · 后期元素 / 转场 / 时机重构

- 日期：2026-06-23
- 范围：`workspace/` 工作台的「画面设计」模块（`#标题/design`）
- 状态：设计定稿，待写实现计划

## 背景与问题

设计台每镜的「✂️ 剪映后期」当前是 **5 个固定单值格**（文字/贴纸/特效/动画/转场），每格 = 一个内容输入框 + 一个时机输入框，存成两个分离的单字符串（`shot.post[k]` + `shot.timing[k]`）。音频四格同理（`shot.audio[k]` + `shot.audioTiming[k]`）。

设计师反馈三个痛点（经确认全部成立）：

1. **多文本/多特效加不进去** —— 一个镜只能有一条文字、一个特效。想要「两条字幕、各自不同时机」表达不了。
2. **转场语义错位** —— 转场被塞进每镜的一个格（`shot.post.trans`），但转场本质是「镜→镜之间」的过渡；当前 N 镜有 N 个转场格，最后一镜根本没有「下一镜」可过渡，语义荒谬。
3. **时机使用逻辑有问题**：
   - 只记「出现」、不记「消失/持续」；
   - 时机是自由文字「念到…时出现」，靠人脑对齐口播，不精确；
   - 多条内容被迫共用一个时机框；
   - 每格都强制想一个时机，多数元素其实跟着镜头走即可，太累。

**根因诊断**：痛点 1 和 3 是同一个结 —— 数据模型把「内容」和「时机」拆成两个分离的单值字段。一旦要「多条内容、各自时机」，模型就崩。痛点 2 是转场被错误地塞进了为「有起止区间的元素」设计的格子里。

## 目标

- 后期元素（文字/贴纸/特效/动画）支持**多条**，每条自带**进/出时机**；
- 时机用**可视化的口播文字轴**表达（拖条段 / 点口播词），**默认「跟镜」不用填**；
- 转场从后期元素里**剥离**，建模为**镜间槽**（N 镜 → N−1 个），逐槽独立填写；
- 音频**本次不动**（异质：BGM 铺底 / 变声是效果 / 只有音效才真需要时机，套同模型会别扭，留待后续）。

## 数据模型

把每镜 flat 的 `post` / `timing` 重构成**后期元素列表 + 独立转场**：

```js
shot = {
  no, line, subtitle,                         // 镜号 / 口播 / 字幕 —— 不变
  subject: { type, assetIds, refs, prompt },  // 画面主体 —— 不变
  post: [                                     // ✦ 后期元素：原 5 个单值格 → 列表，可多条
    {
      kind: 'text' | 'sticker' | 'fx' | 'anim',
      content: '省钱秘诀',
      range: [3, 7] | null,   // 口播文字轴上的字符区间 [起,止)；null = 跟镜(铺满)
      note: ''                // 备注（迁移时放旧 timing 文字）
    }
  ],
  trans: '淡入淡出' | null,    // ✦ 转场：本镜 → 下一镜（末镜恒 null，不渲染槽）
  audio: {...}, audioTiming: {...}            // 音频 —— 本次不动，原样保留
}
```

### `range` 的定义

- `[startChar, endChar)` **半开区间**，索引基于 `shot.line`（口播原文）。
- 例：`line = "这有个省钱秘诀关键其实…"`，元素「省钱秘诀」的 `range = [3, 7)`。
- `null` = **跟镜**（等效 `[0, line.length)`，铺满整条轴，不用想时机）。
- **点口播词** 与 **拖条段两端** 改的是同一份 `range`，两种操作等价：
  - 点口播词 → 先选中该元素条段的「进端」或「出端」（进入编辑态高亮），再点口播某个词，即把该端定位到这个词的字符区间；
  - 拖条段两端 → 直接拖出字符区间（主交互，最直观）；
  - 双击条段 → 重置为 `null`（跟镜）。
- **`line` 为空时**（理论上不应出现，口播来自审核定稿断句）：轴不渲染，元素 `range` 恒 `null`（只能跟镜）。

### `trans` 的定义

- 存 `shot[i].trans`，语义「镜 i → 镜 i+1」的转场。
- UI 画在镜 i 与镜 i+1 **两块镜卡之间**。
- 末镜（第 N 镜，`i = N-1`）的 `trans` 恒为 `null`，不渲染槽。
- 因此 N 镜 = **N−1 个有效转场槽**，逐槽独立填写，**无全局默认**。
- 转场**不进** `post` 列表、**没有** `range`（它是瞬间过渡，无起止区间）；可带可选时长。

### schema 版本

新增 `design.schemaVer` 字段标识数据结构版本：

- 无 `schemaVer` 或 `post` 是对象（非数组） → 旧 schema（flat），加载时迁移；
- `schemaVer >= 2` 且 `post` 是数组 → 新 schema，跳过迁移。

## 旧数据迁移

加载某篇 json 时，若检测到 `design.schemaVer < 2` 或 `shot.post` 不是数组，对每个 `shot` 跑一次迁移，跑完置 `design.schemaVer = 2` 并保存：

| 旧字段 | 去向 |
|---|---|
| `post.text` / `sticker` / `fx` / `anim`（非空） | 各 push 一条 `{ kind, content: 旧值, range: null, note: '' }` |
| `timing.text` / `sticker` / `fx` / `anim`（旧「念到…时」自由文字） | 塞进**对应元素**的 `note`（保住设计师原意图；自由文字无法自动换算成字符区间，仅留作参考） |
| `post.trans`（非空，且 `i < N-1`） | `shot[i].trans = 旧值`（语义「镜 i → 镜 i+1」，位置与旧实现一致） |
| `post.trans`（末镜 `i = N-1`） | 丢弃（末镜无下一镜） |
| `timing.trans` | 丢弃（转场无时机） |
| `audio` / `audioTiming` | 原样保留不动 |

迁移后所有元素 `range` 默认 `null`（跟镜），等设计师在新轴上重新拖/点设区间；旧 timing 意图保留在 `note` 里不丢。

## UI 与交互（电脑端）

每镜卡的新布局——后期元素从「5 个固定格」变成**可增删的元素列表 + 一条多轨口播文字轴**，转场挪到镜卡之间：

```
┌──────────────────────────────────────────────────────────┐
│ 03  [缩略图]  口播：这有个省钱秘诀，关键其实…           │
│              字幕：[省钱秘诀，别错过]                    │
│                                                          │
│ 🖼 画面主体   [未定][选库][AI生成]        （不变）        │
│                                                          │
│ ✂️ 后期元素                          [ ＋ 加一条 ▾ ]      │
│   文字 「省钱秘诀」   ⏱ 念到"秘诀"→"其实"      ✕        │
│   文字 「划重点」     ⏱ 跟镜                    ✕        │
│   特效 「光斑」       ⏱ 跟镜                    ✕        │
│                                                          │
│   口播文字轴  （拖条段改区间 · 点口播词设进/出）         │
│   口播  ｜这有个｜省钱秘诀｜，｜关键｜其实…             │
│   文字①          ████████████               ← 省钱秘诀   │
│   文字② ████████████████████████           ← 划重点(跟镜)│
│   特效  ████████████████████████           ← 光斑(跟镜)  │
│                                                          │
│ 🔊 音频  背景[..] 音效[..] 口播[..] 变声[..]   （不动）  │
└──────────────────────────────────────────────────────────┘
  ▶ 转场到第 4 镜：[ 淡入淡出 ▾ ]    ← 镜间槽，点选转场类型
```

交互清单：

- **＋加一条** → 选类型（文字/贴纸/特效/动画），新增元素默认 `range: null`（跟镜，不用想时机）——直接解决「每格都逼着想时机太累」。
- **每条元素一行**：内容框可编辑；`⏱` 标签显示时机状态（`跟镜` / 锚词预览如 `秘诀→其实`）；`✕` 删除该条。
- **口播文字轴 · 默认常显**（用户已确认）：多轨泳道——口播在最上轨（按词断成可点词块），每条后期元素占一轨、画自己的 `range` 条段。
  - 跟镜（`range: null`）= 整条铺满、浅色；
  - 已设 = 实色段；
  - 同类元素同色，靠轨道纵向堆叠区分，不依赖横向对齐。
  - 拖条段两端改区间、点口播词设进/出、双击条段重置为跟镜。
- **转场槽**：画在两块镜卡之间，点开下拉选转场类型（淡入淡出/缩放/滑动…），可填时长（可选）；末镜之后无槽。

## 导出适配（`workspace/js/export.js`）

- **剪辑指引**（`exportCutGuide`）：当前按 flat 五格渲染，本次改为按 `post` 元素列表 + `trans` 渲染。时机区间**翻译回人话**——`range [3, 7)` 在 `line` 上 `slice` 出「省钱秘诀」，导出成「念到"省钱秘诀"出现 → "其实"消失」；`range: null` 导出成「跟镜（全程）」；`note` 非空时附在末尾作参考。转场单列「转场到第 N 镜：淡入淡出」。
  ```
  第3镜
    后期：
      · 文字「省钱秘诀」  念到"省钱秘诀"出现 → "其实"消失
      · 文字「划重点」    跟镜（全程）
      · 特效「光斑」      跟镜
    转场到第4镜：淡入淡出
  ```
- **生图清单**（`exportShotList`）：只依赖 `subject`（画面主体），后期元素不影响它 → **基本不动**。

## 手机端

多轨轴在窄屏会挤 → 轴改为**横向滚动**（`overflow-x`），词块仍可点设进/出；转场槽、元素列表天然适配。不深入做手机端提效（与既有规格一致：电脑端定稿、手机端待实际操作时再设计）。

## 不做（YAGNI 边界）

- 音频不动（`audio` / `audioTiming` 原样）。
- 不引入秒数 / 真实音频波形（口播时长由剪映 AI 配音决定、设计阶段不可控）。
- 转场无全局默认（逐镜间槽各定）。
- 不做镜间间隔（既有规格里的留接口项）。
